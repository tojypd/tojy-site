 # TOJY 홈페이지 — GitHub이 처음이신 분을 위한 상세 가이드

이 문서는 클릭 단위로 자세히 적었습니다. 순서대로 그대로 따라 하시면 됩니다.

---

## STEP 1. GitHub 계정 만들기 (이미 계정 있으면 건너뛰기)

1. 브라우저에서 **github.com** 접속
2. 우측 상단 **Sign up** 클릭
3. 이메일 주소 입력 → 비밀번호 설정 → 사용자 이름(username) 설정
   - username은 나중에 사이트 주소 일부로도 쓰일 수 있으니 회사명과 비슷하게 (예: `tojy-studio`) 하시는 걸 추천드려요.
4. 이메일 인증 코드가 오면 입력
5. 가입 완료되면 GitHub 메인 화면으로 이동됨

---

## STEP 2. 새 저장소(Repository) 만들기

"저장소"는 이 홈페이지 파일들을 담아둘 폴더라고 생각하시면 됩니다.

1. GitHub 로그인 상태에서 화면 우측 상단의 **+** 아이콘 클릭
2. 드롭다운에서 **New repository** 클릭
3. 아래 항목 입력:
   - **Repository name**: `tojy-site` 입력
   - **Public / Private**: **Public** 선택 (Netlify 무료 연동을 위해 Public 권장)
   - 나머지 옵션(Add README 등)은 전부 **체크하지 않고 그대로 둡니다**
4. 맨 아래 초록색 **Create repository** 버튼 클릭
5. 저장소가 생성되면 비어있는 페이지가 나타나고, 그 안에 파일 업로드 안내 문구가 보입니다

---

## STEP 3. 이 zip 파일을 저장소에 업로드하기

1. 먼저 제가 드린 `tojy-site-github용.zip` 파일을 컴퓨터에 다운로드한 뒤, **압축을 풀어주세요** (우클릭 → 압축 풀기 / 압축 해제)
   - 압축을 풀면 `tojy-site` 라는 폴더가 나옵니다
2. 방금 만든 GitHub 저장소 페이지에서 **uploading an existing file** 이라는 파란색 글자 링크 클릭
   (화면에 안 보이면 상단 **Add file → Upload files** 버튼을 눌러도 동일합니다)
3. 압축 푼 `tojy-site` 폴더를 **열어서 그 안의 내용물 전체**를 선택합니다
   - `src`, `public` 폴더와 `package.json`, `index.html`, `사용가이드.md` 등 파일들이 보일 거예요
   - **주의**: `tojy-site` 폴더 자체가 아니라, 그 **안에 있는 파일/폴더들**을 전체 선택해서 끌어다 놓아야 합니다
4. 그 상태로 웹페이지의 업로드 박스 위로 **드래그 앤 드롭**
5. 파일들이 업로드 목록에 쭉 뜨는 걸 확인 (`node_modules` 폴더는 없는 상태라 용량이 크지 않을 거예요)
6. 페이지 맨 아래로 스크롤 → **Commit changes** 초록 버튼 클릭
7. 업로드가 끝나면 저장소 페이지에 폴더/파일 목록이 보이게 됩니다 — 완료!

---

## STEP 4. Netlify로 실제 사이트 배포하기

1. 브라우저에서 **netlify.com** 접속 → **Sign up** (GitHub 계정으로 가입하면 다음 단계가 더 빠릅니다 → **Sign up with GitHub** 클릭)
2. GitHub 로그인 창이 뜨면 로그인하고 **Authorize netlify** 클릭 (Netlify가 내 GitHub 저장소를 볼 수 있도록 허용하는 것)
3. Netlify 대시보드에서 **Add new site** 버튼 클릭 → **Import an existing project** 선택
4. **Deploy with GitHub** 클릭
5. 저장소 목록에서 아까 만든 **tojy-site** 선택
   - 목록에 안 보이면 **Configure the Netlify app on GitHub** 링크를 눌러서 저장소 접근 권한을 허용해주세요
6. Build settings 화면이 나오는데, 아래처럼 되어있는지 확인 (Vite 프로젝트라 자동으로 채워질 가능성이 높습니다):
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
   - 비어있으면 직접 위 값을 입력해주세요
7. **Deploy site** 버튼 클릭
8. 화면에 진행 바가 뜨고 1~3분 정도 기다리면 배포 완료
9. 상단에 `random-name-123456.netlify.app` 같은 주소가 생성됩니다 → 클릭하면 실제 사이트가 열립니다 🎉

---

## STEP 5. 사이트 이름(서브도메인) 예쁘게 바꾸기 (선택)

1. Netlify 사이트 대시보드에서 **Site settings** 클릭
2. **Change site name** 버튼 클릭
3. 원하는 이름 입력 (예: `tojy-studio`) → 저장
4. 이제 주소가 `tojy-studio.netlify.app` 으로 바뀝니다

---

## STEP 6. 사진 교체하는 법 (앞으로 제일 자주 하실 일)

1. GitHub 저장소 페이지 접속 → `public` 폴더 클릭 → `images` 폴더 클릭 → `works`(또는 `team`, `hero`) 폴더 클릭
2. 교체하고 싶은 파일 이름 확인 (예: `w1.jpg`가 어떤 작품인지는 `src/app/data.ts` 파일 열어서 확인)
3. 그 폴더 화면에서 **Add file → Upload files** 클릭
4. 새 사진 파일을 올리는데, **파일명을 기존과 똑같이** (`w1.jpg`) 맞춰서 업로드
   - 다른 이름으로 올리면 기존 파일이 덮어써지지 않고 새 파일이 추가만 되니, 반드시 이름을 맞춰주세요
5. 맨 아래 **Commit changes** 클릭
6. 30초~2분 기다리면 Netlify가 자동으로 재배포합니다 (Netlify 대시보드의 **Deploys** 탭에서 진행 상황 확인 가능)

---

## STEP 7. 새 작품 추가하는 법

1. 사진을 `public/images/works/` 폴더에 새 파일명으로 업로드 (예: `w29.jpg`)
2. `src/app/data.ts` 파일 클릭 → 우측 상단 **연필 아이콘(Edit this file)** 클릭
3. `WORKS` 배열 맨 끝, 마지막 `}` 바로 앞에 아래처럼 추가:

```ts
{
  id: "w29",
  title: "새 작품 제목",
  network: "방송사/플랫폼",
  year: "2026",
  category: "카테고리명",
  imageUrl: "/images/works/w29.jpg",
  description: "한 줄 설명",
  producer: "담당 대표님 이름",
},
```

4. 우측 상단 **Commit changes** 클릭 → 자동 재배포

---

## STEP 8. 지금 확인이 더 필요한 항목들

`src/app/data.ts` 파일을 열면 맨 위 주석에 상태가 정리돼 있는데, 아직 남은 것들:
- `트라이앵글`, `무명전설 디너쇼`, `쏠 트레블J카드 광고`, `교환왔수다` — 정확한 설명 문구(description)는 비워뒀습니다. 채워주시면 반영해드릴게요.
- 위 6개 작품 + 이웃집 찰스는 아직 실제 사진이 없어서 "사진 교체 필요" 표시된 이미지가 나옵니다. STEP 6 방법으로 교체해주세요.

---

## STEP 9. 나중에 실제 도메인을 구매하면

Netlify의 **Site settings → Domain management → Add a domain**에서 구매한 도메인을 입력하고, 도메인을 산 곳(가비아, 후이즈 등)에서 안내하는 네임서버/DNS 값을 등록하면 며칠 내로 연결됩니다. `.netlify.app` 서브도메인은 계속 무료로 함께 유지됩니다.
